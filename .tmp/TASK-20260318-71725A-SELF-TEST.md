## Issues List (Prioritized)

### Medium

1. **Cross-tenant metadata leakage risk in audit integrity endpoint**
   - Evidence: `pure_backend/main.py:1716` queries all `AuditLog` rows globally without organization filter.
   - Impact: tenant A can infer total audit volume/chain integrity influenced by tenant B activity.
   - Minimal improvement: restrict query to `AuditLog.organization_id == x_org_id` or clearly define/guard global-auditor-only semantics with dedicated super-admin scope.
   - Repro: call `/governance/audit/integrity` from one org after generating audit events in another org; observe global `total_records` growth.

2. **Backup/archive governance is record-level simulation, not daily automated full backup execution**
   - Evidence: endpoints directly set `status="DONE"` and write descriptive text (`pure_backend/main.py:1455`, `pure_backend/main.py:1478`) without actual backup artifact generation/scheduler loop.
   - Impact: operational resilience/compliance expectations may be overstated in production.
   - Minimal improvement: add periodic scheduler worker + backup artifact path/checksum manifest + restore verification endpoint.
   - Repro: trigger `/governance/backup`; verify only DB task record returned.
